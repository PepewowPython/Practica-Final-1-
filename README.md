# Práctica Final - Rutas Inseguras

- `index.js`: Punto de entrada del proyecto.

- `SEMI-PRACTICA-FRONTEND.pdf`: Documento de práctica.

## Flujo de trabajo recomendado

1. Haz `git pull` antes de empezar a trabajar.
2. Crea una rama nueva para tu tarea:
   - `git checkout -b mi-parte`
3. Trabaja en los archivos que te correspondan.
4. Haz `git add` y `git commit -m "Mensaje claro"` frecuentemente.
5. Sube tu rama:
   - `git push origin mi-parte`
6. Si hay cambios en el remoto, actualiza tu rama:
   - `git checkout mi-parte`
   - `git pull --rebase`

## Coordinación con el equipo

- Es buena idea que cada persona comente su parte antes de empezar.
- Si varios editan el mismo archivo, hace falta comunicarse para evitar conflictos.

## Qué puede hacerse en `index.js`

- Agregar lógica de la aplicación o componentes de la interfaz.
- Usar `index.js` como base para pruebas sencillas de JavaScript.
- Si el equipo quiere, se puede dividir en más archivos para cada módulo.
